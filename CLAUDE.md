## Development Guidelines

### Editing Content

All content is plain Markdown. Changes take effect on the GitHub profile immediately after pushing to the default branch (`main`).

### What to Avoid

- Do not add build tooling, package managers, or dependencies — this is a Markdown-only profile repo.
- Do not remove the HTML comment block in README.md — it serves as a template reference for the owner.
- Do not change the repository's fundamental purpose (GitHub profile display).

### When Making Changes

1. Edit Markdown files directly.
2. Commit with a clear, concise message describing the change.
3. Push to the appropriate branch.
